Q3: Diffence between local storage and session storage

Ans : We know two storage systems, Local and session storage, Local storage stores info until it is removed where session storage is only available for a session of a tab. If Once it is closed, the information will be deleted, but local storage info will not be deleted if the browser or tab is closed, the information will still be there when the browser reopens. Local storage info is also available to multiple tabs. In terms of security protection, local storage is not effective, as javaScript that an attacker injects via XSS runs on the same origin as the rest of the browser application code. An attacker can easily retrieve the value of a secret named with a single line but session storage is less risky than local storage. We can store data in local storage by writing local storage.setItem() and by localStorage.removeItem() We can remove items from local storage.
